# MIRROR Bot

Telegram-бот для зеркалирования каналов, чатов и форумов (включая ветки).

Бот управляет настройками через кнопки, а worker копирует сообщения от имени Telegram-сессии пользователя.

## Что умеет

- Подключение пользовательских Telegram-сессий (API_ID/API_HASH + авторизация)
- Зеркала для:
  - каналов
  - групп/чатов
  - форумов с темами
- Фильтрация по ключевым словам или режим без фильтра
- Копирование текста и медиа (фото/видео/документы)
- Форматирование автора сообщения перед отправкой
- Редактор веток форума:
  - подключение тем из исходного форума
  - внешние ветки (источник из другого чата/канала/форума)
- Автообнаружение новых тем в исходном форуме:
  - бот присылает предложение подключить новую ветку
  - кнопки: `Подключить ветку` / `Скрыть`

## Важное поведение

- Это **копия**, а не Telegram-forward:
  - сообщение отправляется как новое в целевой чат
  - автор источника не получает уведомление о пересылке
- Секреты и рабочие данные не должны попадать в git:
  - `.env` (игнорируется)
  - `data/` (игнорируется)
  - базы и сессии (игнорируются)

## Быстрый старт (Docker)

### 1) Клонирование

```bash
git clone <repo-url> mirror-bot
cd mirror-bot
```

### 2) Переменные окружения

```bash
cp .env.example .env
```

Откройте `.env` и заполните:

```env
BOT_TOKEN=your_bot_token_from_botfather
```

Получить токен: [@BotFather](https://t.me/BotFather)

### 3) Запуск

```bash
docker compose up -d --build
```

### 4) Проверка

```bash
docker compose ps
docker compose logs -f bot
docker compose logs -f bridge
```

## Локальный запуск (без Docker)

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Терминал 1
python -m src.bot

# Терминал 2
python main.py
```

## Первичная настройка в Telegram

1. Напишите боту `/start`.
2. Нажмите `📲 Подключить аккаунт`.
3. Введите `API_ID` и `API_HASH` (из [my.telegram.org](https://my.telegram.org)).
4. Пройдите авторизацию Telegram-сессии.

После этого можно создавать зеркала.

## Создание зеркала

Через кнопку `✨ Создать зеркало`:

1. Выбор аккаунта
2. Выбор типа источника: канал / группа / форум
3. Указание источника (поддерживаются `@username`, `t.me`, `web.telegram.org`, `tg://`, numeric id)
4. Выбор фильтра

Для форумов:
- показывается список тем
- `General` обязательна
- выбор тем можно изменить позже в редакторе веток

## Редактор веток форума

В карточке forum-зеркала:
- `Управление ветками`
- `Добавить ветки из форума`
- `Подключить внешний источник`

Если в исходном форуме появилась новая тема:
- worker обнаружит её
- в бот придёт предложение подключения
- после `Подключить ветку` создаётся тема в целевом форуме и включается маршрутизация

## Структура проекта

```text

```

## Troubleshooting

- `BOT_TOKEN is required in .env`
  - Проверьте, что `.env` существует и заполнен.
- Конфликт бота (`TelegramConflictError`)
  - Убедитесь, что запущен только один экземпляр `src.bot`.
- Новая тема форума не подключилась
  - Проверьте уведомление в боте и нажмите `Подключить ветку`.
- Нет маршрутизации сразу после изменения правил
  - Worker использует сигнал `data/route_reload.signal`; обычно подхват происходит автоматически.

## Безопасность

- Не коммитьте `.env`.
- Не коммитьте `data/`.
- Не публикуйте session strings.
- Для публичного деплоя используйте отдельного бота и отдельные API-ключи.
